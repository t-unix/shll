## Reproduce

```sh
# host
orb create debian shll-test
orb -m shll-test exec -- bash /Users/$USER/Development/shll/bootstrap/test/inside-vm-bootstrap.sh

# inside VM (orb -m shll-test shell)
sudo apt-get install -y tmux fish fzf neovim pass gnupg
# Render templates (currently driven by `claude /shll-install` once authenticated;
# see ../orbstack.md for the manual fallback used to generate these screenshots.)
```

## Notes

- Steps 1–6 require no Claude Code authentication inside the VM and exercise the
  pieces that are 100% manifest-driven (OS detection, tool resolution, apt install).
- Step 7–8 were generated by running a minimal Python template renderer inside
  the VM that mirrors `lib/render.md` — substituting `{{ cmd_or_super }}` and the
  defaulted `user.terminal.*` variables. The full install skill (`claude /shll-install`)
  does the same render plus backup-existing-target + scrubbed assertion.
- WezTerm/Alacritty weren't apt-installed in this round (they need pre-step PPAs);
  the rendered configs land regardless and would be picked up the next time the
  binaries are present.
